## How to run

```bash
# First time only (creates the test DB and pushes schema)
docker run --name chao-pg -e POSTGRES_USER=chao -e POSTGRES_PASSWORD=chao_secret_2025 -e POSTGRES_DB=chao_os -p 5432:5432 -d --restart unless-stopped postgres:16-alpine
docker exec chao-pg psql -U chao -d postgres -c "CREATE DATABASE chao_os_test;"
npm run test:db:push
Copy-Item .env.test.example .env.test

# Run all tests
npm run test:run

# Run with coverage
npm run test:coverage

# Run a single test file
npm run test:run -- tests/api/auth-guard.test.ts

# Watch mode (re-runs on file change)
npm test
```

## Architectural decisions

| Decision | Rationale |
|---|---|
| Vitest 3.x (not 4.x) | V4 has unresolved Next.js 14 subpath import bugs |
| `nextjsSubpathFix` Vite plugin | Next 14.2.x has no `package.json#exports` field for `next/server` / `next/headers`. Vite can't resolve them. Plugin intercepts `resolveId` and points to the actual `.js` file. |
| Direct route handler invocation (not HTTP fetch) | Faster (no server start), deterministic, and tests the route logic in isolation. Mock `auth()` for session, mock `email.ts` for fire-and-forget emails. |
| Real credentials flow for TEST 1 | The credentials provider IS the auth. Testing the extracted `authorizeCredentials` exercises the real bcrypt + DB lookup path. |
| `fileParallelism: false` + `singleFork: true` | Shared Postgres DB cannot handle parallel writes from multiple test files. Sequential = deterministic, no flakiness. |
| `setup.ts` hard-fails on wrong DATABASE_URL | Prevents accidentally running tests against dev/prod (catastrophic). |
| 5 tests = minimum viable suite | Covers the 4 most critical business operations (auth, routing, lead conversion, invoice lifecycle) + 1 regression test (seed). Expanding to 30+ tests is a separate sprint. |
